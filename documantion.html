Attendance App — Quick Guide

What it does, where to go, and examples to try.
Repo: mnuatendancemnumdelsakkatech

Core Pages

  • Sign Up — create a student account and enroll in modules (password required).
  • Sign In — submit attendance codes (location + code).
  • Student Dashboard — view your enrolled modules, per-lecture attendance and missed lectures.
  • Admin Panel — manage modules, lecture schedules, enroll/unenroll students, export CSV.
  • Database helpers — central PDO helpers and schema (SQLite).

Features

Module CRUD
Create modules and lecture schedules with dates (lecture codes stored in lecture_codes).
Student Signup
Students register, set a password, and enroll in modules. Passwords are bcrypt-hashed in attendance.password_hash.
Attendance
Students submit responses (code + geo). Responses are stored in attendance.responses_json and counted per lecture.
Date-aware
Only lectures with a date <= today are counted as "past" for absence tallies; future lectures are shown but not counted as absences.
Dismissal Rule
Dismissal is based on absences across all module lectures: dismissed when (absences / total lectures) > 25%. Exact 25% is not dismissed.
CSV Export
Admin can export attendance with missed lecture details from the admin panel.

Examples

Sign up & submit attendance

  1. Open signup.php and create a student with academic number 2021001, password and enroll in a module (e.g. CORE).
  2. As that student, open signin.php and submit a valid lecture code for a module while in range.
  3. View dashboard.php to see the response listed and whether the lecture is marked attended.

Admin export

<Admin > Export CSV → Downloads a file with missed lecture details and counts.

Troubleshooting & Notes

  • If a student is listed as dismissed, check their module total_all, attended_all, and absent_all in the admin details to confirm the absence ratio.
  • Lecture date parsing accepts YYYY-MM-DD; invalid dates are treated as blank (future/past logic depends on stored date).
  • To reset responses: use the admin "Clear responses" action (available in admin panel).
